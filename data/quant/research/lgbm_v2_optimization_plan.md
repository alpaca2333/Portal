## 二、六大核心问题与改进方向

### 问题一：换手率极高（年化 ~769%），交易成本侵蚀严重 🔴 P0

**现象**：平均每期买入 40.3 笔、卖出 34.1 笔，每两周几乎换掉一半持仓。

**根因分析**：
- `top_pct=0.05`（前5%）选出约 50~100 只股票，但双周调仓频率下 ML 分数排名波动很大
- `buffer_sigma=0.3` 的缓冲带太弱，不足以稳定持仓
- 22 个特征中有大量短期因子（`rev_10`、`ret_5d_std`、`volume_chg`），导致分数双周间剧烈变化

**改进方案**：

| 方案 | 具体操作 | 预期效果 |
|------|---------|---------|
| A. 增大缓冲带 | `buffer_sigma` 从 0.3 → 0.5~0.8 | 减少 20~30% 换手 |
| B. 降低调仓频率 | `BW` → `M`（月度），或引入最小持仓期 | 换手率减半 |
| C. 限制单期换手 | 每期最多换掉 30% 持仓 | 平滑换手 |
| D. 减少短期噪声因子 | 移除或降权 `rev_10`、`ret_5d_std` | 分数更稳定 |

**实现优先级**：A → B → C → D

---

### 问题二：盈亏比不到 1（0.95），大亏期拖累严重 🔴 P0

**现象**：胜率 57.7% 看似不错，但赢的时候平均赚的比输的时候亏的少。2024-02-05 单期 -18.15%（基准仅 -2%）是灾难性事件。

**根因分析**：
- 策略偏向小盘股（`mv_pct_upper=0.85` 覆盖大量小盘），在流动性危机中集中踩踏
- 没有任何风控模块，极端行情下无法自我保护
- 等权分配导致高波动股票对组合影响过大

**改进方案**：

| 方案 | 具体操作 | 预期效果 |
|------|---------|---------|
| A. 动态仓位管理 | 当市场波动率急升时自动降仓位 | 最大回撤 -29% → -20% |
| B. 收紧市值下限 | 极端行情下动态收紧 `mv_pct_upper` | 避免微盘股踩踏 |
| C. 单期止损 | 组合单周跌幅超阈值触发减仓 | 截断尾部风险 |
| D. 风险平价加权 | 按个股波动率倒数加权 | 降低高波动股影响 |

**实现优先级**：A → B → D → C

---

### 问题三：2018 和 2020 年超额为负，模型泛化能力不足 🟡 P1

**现象**：
- 2018 年（熊市）：超额 -0.08%/期，胜率 44%
- 2020 年（核心资产牛市）：超额 -0.10%/期，胜率 42%

**根因分析**：
- 3 年滚动窗口训练，但 2018 年初模型用的是 2015-2017 数据（牛转熊），学到的模式失效
- 模型没有"市场状态"感知能力，无法区分牛市/熊市/震荡市

**改进方案**：

| 方案 | 具体操作 |
|------|---------|
| A. 加入市场状态特征 | 指数 MA20/MA60 比值、市场波动率、涨跌家数比 |
| B. 自适应训练窗口 | 高波动期缩短窗口（1.5年），低波动期延长（4年） |
| C. 多模型集成 | 训练多个不同窗口长度的模型，加权平均预测 |

---

### 问题四：标签设计过于简单（单期 forward return） 🟡 P1

**现象**：双周 forward return 的截面排名作为标签，信噪比低。

**根因分析**：
- 双周收益噪声极大
- 没有考虑风险调整后的收益
- 排名标签丢失了收益幅度信息

**改进方案**：

| 方案 | 具体操作 | 预期效果 |
|------|---------|---------|
| A. 多期标签平滑 | 用未来 1 个月累计收益作为标签 | 降低噪声 |
| B. 风险调整标签 | 用 `return / volatility` 作为标签 | 偏好稳定收益 |
| C. 分类目标 | 标签分 5 档，用 LambdaRank 替代回归 | 更适合排序任务 |

---

### 问题五：特征工程存在冗余和缺失 🟡 P1

**冗余问题**：
- 3 个动量因子（`mom_12_1`、`mom_3_1`、`mom_6_1`）高度相关
- `rvol_20` / `ret_5d_std` / `high_low_20` 都在度量波动性
- `inv_pb` / `pe_ttm` 都是价值因子

**缺失因子**：
- ❌ 行业相对动量（个股动量 - 行业平均动量）
- ❌ 波动率调整动量（`mom / rvol`）
- ❌ 市场状态因子（指数趋势、市场情绪）
- ❌ 市场 beta（个股对市场的敏感度）
- ❌ 资金流向因子（北向资金、主力净流入 — 受数据源限制）

**改进方案**：

| 方案 | 具体操作 |
|------|---------|
| A. 加入行业相对动量 | `stock_mom - industry_avg_mom` |
| B. 加入波动率调整动量 | `mom / rvol`，提高信噪比 |
| C. 加入市场 beta | 个股对市场的敏感度 |
| D. 特征重要性分析 | 移除贡献低的冗余特征 |

---

### 问题六：选股和权重分配过于粗糙 🟢 P2

**现象**：完全等权分配，没有利用 ML 分数的信号强度。

**改进方案**：

| 方案 | 具体操作 | 预期效果 |
|------|---------|---------|
| A. 信号强度加权 | 按 ML 分数排名分配权重 | 夏普 +0.03~0.08 |
| B. 风险平价加权 | 按个股波动率倒数加权 | 降低组合波动 |
| C. 行业中性化 | 各行业权重与基准一致，只做行业内选股 | 降低行业偏离风险 |
| D. 组合优化 | 加入协方差约束，控制组合整体风险 | 更优风险收益比 |

## 三、改进优先级总结

| 优先级 | 改进项 | 预期影响 |
|--------|--------|---------|
| 🔴 P0 | 降低换手率（增大 buffer / 改月度调仓 / 限制换手） | 夏普 +0.05~0.10 |
| 🔴 P0 | 极端风险控制（动态仓位 / 收紧小盘暴露） | 最大回撤 -29% → -20% |
| 🟡 P1 | 标签优化（多期平滑 / LambdaRank） | 夏普 +0.05~0.15 |
| 🟡 P1 | 特征增强（行业相对动量 / 市场状态 / 去冗余） | 超额 +1~3%/年 |
| 🟢 P2 | 权重优化（信号强度加权 / 风险平价） | 夏普 +0.03~0.08 |
| 🟢 P2 | 模型增强（多窗口集成 / 自适应窗口） | 泛化能力提升 |

## 四、建议实施路径

```
Phase 1 (V4.1) — 降本控险
├── buffer_sigma: 0.3 → 0.6
├── 调仓频率: BW → M
├── 加入最大单期换手限制 (30%)
└── 加入市场波动率感知的动态仓位

Phase 2 (V4.2) — 信号增强
├── 标签: 双周 forward return → 月度 forward return
├── 新增特征: 行业相对动量、波动率调整动量、市场 beta
├── 特征重要性分析 → 移除冗余
└── 目标函数: regression → LambdaRank

Phase 3 (V5) — 组合优化
├── 等权 → 信号强度加权 + 风险平价
├── 行业中性化约束
├── 多窗口模型集成
└── 自适应训练窗口
```
